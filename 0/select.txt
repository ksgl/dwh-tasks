Вопрос:
У Вас ситуация: таблица T, несколько миллиардов записей. В колонке А - 5 разных возможных значений, в
колонке B - 50 разных возможных значений, в колонке C - 100 000 разных возможных значений. Распределение
количества строк по знaчениям A, B, C - равномерное. О производительности INSERT, UPDATE, DELETE -
беспокоиться не надо. Но надо сделать макс. возможно хорошие условия для

SELECT * FROM T WHERE A=? AND B=? AND C=?

Какой индекс(индексы) сделаете? С коротким обьяснением почему такой индекс.

Возможные варинты ответа:
1) Вариативность данных в колонках var(A) < var(B) < var(C) (по условию), но записей в таблице >= 1b.
Таким образом, селективность колонки (обозначим sel, количеством записей в таблице с одинаковыми значениями) sel(A) < sel(B) < sel(C).
Когда записей с одинаковым значением мало — селективность высокая. Такие колонки необходимо использовать первыми в составных индексах.
Тогда эффективным окажется индекс (C, B, A).

2) Согласно статье https://www.slac.stanford.edu/pubs/slacpubs/16250/slac-pub-16460.pdf, Hash index является хорошей стратегией индексирования, так как в запросе просходит сравнение на равенство "=".
